NAME

       ezmlm-warn - send out bounce warnings


SYNOPSIS

       ezmlm-warn [ -dD ][ -t timeout ] dir


DESCRIPTION

       ezmlm-warn sends out warning messages for the mailing list
       stored in dir.

       ezmlm-warn scans dir/bounce/ for bounce messages  received
       by  ezmlm-return.   If  it  sees a distribution bounce for
       box@domain received more  than  ten  days  ago,  it  sends
       box@domain  a  list  of  all  the  message  numbers missed
       recently, and deletes the bounce.  If it  sees  a  warning
       bounce  for box@domain received more than ten days ago, it
       sends box@domain a probe, and deletes the bounce.

       ezmlm-warn will not send a warning or probe to an  address
       that is not currently a subscriber.


OPTIONS

       -d     process  bounces  for  the digest list, rather than
              for the main list.  Digest list bounces are  stored
              in dir/digest/bounce/ rather than in dir/bounce/.

       -D     (Default.)  Process bounces for the main list.

       -t timeout
              Bounces  received  more  than  timeout days ago are
              processed. This overrides the default  of  approxi-
              mately  10  days  and  may  be useful for very busy
              lists.
